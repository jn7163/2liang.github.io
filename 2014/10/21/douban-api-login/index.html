使用豆瓣帐号登录

“用豆瓣账号登录”允许访问你网站的用户使用豆瓣账号登录。

你的网站使用此功能后，可以让用户无需注册流程直接登录，降低使用门槛，让你的网站轻松获得新用户，提升用户登录率。

 

点击戳进官方文档

登录流程

第一步：申请应用

在申请应用时，有一项【回调地址】需要说明一下，让用户使用第三方账号登陆的原理是：通过授权，获取到用户的个人信息，将获取到的信息存储到数据库中即可。所以，回调地址就是处理你获取到的数据的。

申请成功后，一定要记得添加测试用户，至于怎么添加，申请应用后，你就知道了；还有就是要申请API权；最后提交审核即可。

当申请应用成功后，官方会给我们两个值：

  • API Key：064611c*6b80c1319a
  • Secret：51d18**2a219

第二步：获取authorization_code

因为获取authorization_code使用GET方式提交，所以只需要直接访问即可。

用户点击“用豆瓣账号登陆”按钮，弹出授权页面代码：

<a href=”https://www.douban.com/service/auth2/auth?client_id=064611c*************6b80c1319a&redirect_uri=http://****.sinaapp.com/login.php&response_type=code&scope=douban_basic_common,book_basic_r,book_basic_w"><img src=”./images/douban.png”  alt=”登陆”></a>

问题主要在于构造授权地址，

参数：

参数名称

参数说明

client_id

必选参数，应用的唯一标识，对应于APIKey

redirect_uri

必选参数，用户授权完成后的回调地址，应用需要通过此回调地址获得用户的授权结果。此地址必须与在应用注册时填写的回调地址一致。

response_type

必选参数，此值可以为 code 或者 token 。在本流程中，此值为 code

scope

可选参数，申请权限的范围，如果不填，则使用缺省的scope。如果申请多个scope，使用逗号分隔。

state

可选参数，用来维护请求和回调状态的附加字符串，在授权完成回调时会附加此参数，应用可以根据此字符串来判断上下文关系。



参考即可。

其中scope就是你所申请的API权限，具体值可以在API权限申请页面查看。

如果出现报错，可以在官方文档中查找对应报错原因。

我在获取authorization_code时，出现了113错误，说是缺少参数，查证后并无缺少参数，最终解决的方法竟然是将代码改成UTF-8格式

授权成功后，系统会将authorization_code以GET方式传递给回调地址，所以只需要在回调地址文件中获取这个GET参数的值即可。

第三步：获取access_token

因为获取access_token需要POST请求，所以需要使用到CURL，各种参数跟第二步的基本一致，直接贴上代码：

header(“Content-Type: text/html;charset=utf-8”);
    $data = array(
        ‘client_id’ => “064611c*6b80c1319a”,
        ‘client_secret’ => “51d18**2a219”,
        ‘redirect_uri’ => “http://****5.sinaapp.com/login.php“,
        ‘grant_type’ => “authorization_code”,
        ‘code’ => $_GET[‘code’],
        );
    $url = “https://www.douban.com/service/auth2/token“;
    $ch = curl_init();
    curl_setopt($ch, CURLOPT_URL, $url);
    curl_setopt($ch, CURLOPT_POST, 1);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
    curl_setopt($ch, CURLOPT_POSTFIELDS,$data);
    $return = curl_exec($ch);
curl_close($ch);

此时，你就可以获取到一个json数据，如下：

{
  "access_token":"a14af********f6ff4",
  "expires_in":3920,
  "refresh_token":"5d633d1************24803ec",
  "douban_user_id":"12****1"
}
