在SAE上安装Typecho博客系统

发表于 2014-04-12   |   分类于 爱撸码   |     |    阅读次数

Sina App Engine（以下简称SAE）是新浪研发中心于2009年8月开始内部开发，并在2009年11月3日正式推出第一个Alpha版本的国内首个公有App Engine，SAE是新浪云计算战略的核心组成部分。
作为穷屌丝，买不起服务器也就只能选用SAE了，在SAE上搭建了个Typecho博客系统，中间出现了一点问题，下面分享下安装过程，以备后用。

下载typecho源代码：

http://blog5219.u.qiniudn.com/build.zip

如果你要在官方去下载，记得不要下载那个原版，官方提供了一个SVN版本的typecho源码，下载这个，不然在SAE中安装的时候，会出现没有可写权限，无法进行下一步。我将typecho的数据库也放在了里面，下载后，将数据库也就是.sql文件取出来，后面会用到。

想在SAE上搭建博客，必须得现拥有个SAE账户吧，申请后，创建个应用，开发语言选择PHP即可，然后创建个版本，把代码上传上去，推荐使用SVN管理代码，至于SVN怎么使用，度娘可以告诉你，而且SAE官方也给了使用方法，用起来挺简单的，而且超级给力。现在用到数据库文件了，用记事本或者notepad++打开，将第130行修改为你的SAE域名（修改SAE域名的时候请不要带上你的版本号，以免后期会出现因为重定向而造成的无法进入后台），然后通过phpmyadmin导入数据库。

接着进入你的域名，开始正式安装Typecho博客系统，直接访问会跳转到安装页面，然后直接下一步,会出现：

到了这里，你可以使用SVN在本地新建文件config.inc.php，然后将这里的配置文件内容全部复制到你的新建php文件中，然后上传到SAE服务器，上传完成后，点击上传完毕即可。亦或者直接使用SAE提供的代码在线编辑新建并且将代码复制进去即可。这里就安装完成了。这个一开始安装后，是官方随机给出的密码，记得先将密码修改了。

问题汇总（遇到问题，还会有更新）：

1、出现无读写权限，无法进行下一步

出现无读写权限，是因为你下载的typecho博客程序的原因，应该下载SVN版本的typecho博客程序，并且将.conf(在我提供的源码包里面有)文件添加进去，推荐：直接在我给的链接下载，这个是我修改好的。

2、出现重定向错误

如果在进入后台的时候，出现重定向错误，那就是应为你在修改sql数据库的时候，在你的SAE地址上加上了版本号，只要将版本号去掉就可以了。（修改方式如问题4，因为你已经将数据库上传上去了，所以直接修改130行就又得先删除数据库然后重新导入，略显不方便）

3、绑定域名问题

搭建个博客，大家肯定希望弄上自己的独立域名吧。SAE给我们提供了这个进入应用后，选择应用设置，会出现绑定独立域名的设置，跟着提示来就可以了。

点击绑定后，会出现下面的让你绑定指定二级域名，只要跟着来就可以了。

在你的域名服务器中解析成功后，点击确认，SAE会自行审核，审核成功就可以了。

4、访问博客链接自动跳转至SAE域名

其实这个就是.sql文件的问题，一开始不是让大家将那个url改为自己的SAE链接，那是因为没有绑定域名，现在绑定域名了，自然就要修改成我们的独立域名了，进入Mysql数据库，然后使用SAE提供的phpmyadmin进行修改（管理Mysql），看图：
